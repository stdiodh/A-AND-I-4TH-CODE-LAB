당신은 A&I 백엔드 커리큘럼용 실습 레포와 문서를 설계하는 Codex입니다.

지금부터는 전체 커리큘럼 중 **09. 배포와 실행 환경** 시퀀스만 다룹니다.
다른 시퀀스 내용은 절대 섞지 말고, 이번 요청에서는 09 시퀀스에 필요한 레포 구조, 학생용 starter 코드, 정답 가이드, 이론 문서만 정확하게 만드세요.

## 1. 이번 작업의 목표

이번 시퀀스의 목적은 이전 시퀀스들에서 만든 애플리케이션을 바탕으로,
학생이 아래 흐름을 직접 연결하면서 **로컬 실행과 서버 실행의 차이**, **Docker의 역할**,
**환경변수와 로그가 왜 필요한지**를 이해하게 만드는 것입니다.

- 로컬에서 돌던 앱을 Docker 이미지로 만든다
- 컨테이너로 실행한다
- 서버 환경에서 실행한다
- 환경변수를 분리해서 적용한다
- 서버 로그를 보고 문제를 확인한다

즉, 이번 시퀀스는 AWS 운영 전체를 깊게 배우는 단계가 아닙니다.
학생이 **배포는 단순 복사가 아니라 실행 환경을 맞추는 일**이고,
**운영 환경에서는 환경 분리와 로그 확인이 왜 중요한지**를 입문 수준에서 이해하게 만드는 단계입니다.

## 2. 이번 시퀀스에서 학생이 완성해야 하는 최종 상태

학생은 이 시퀀스를 마친 뒤 아래를 할 수 있어야 합니다.

1. Docker를 왜 쓰는지 설명할 수 있다.
2. 로컬 실행과 서버 실행의 차이를 말할 수 있다.
3. Dockerfile이 어떤 역할을 하는지 설명할 수 있다.
4. 환경변수를 왜 코드와 분리해야 하는지 설명할 수 있다.
5. profile 또는 운영 설정이 왜 필요한지 이해할 수 있다.
6. 서버에서 문제가 생겼을 때 로그를 어디서 확인해야 하는지 말할 수 있다.

- 보안 민감 정보는 실제 값이 아니라 예시 구조로만 보여주세요.
- 문서는 학생용 복습과 강사용 PPT 준비까지 같이 고려해서 쓰세요.

## 10. TODO 작성 규칙

TODO는 반드시 순서형 힌트로 작성하세요.

좋지 않은 예:
```dockerfile
# TODO: 도커 설정
````

좋은 예:

```dockerfile
# TODO 1. 애플리케이션 jar 파일 경로를 맞추세요.
# TODO 2. 컨테이너가 실행할 기본 명령을 작성하세요.
# TODO 3. 지금 단계에서는 복잡한 멀티 스테이지 최적화보다 '실행 흐름 이해'가 우선입니다.
```

환경변수 예:

```yaml
# TODO 1. 운영 환경에서 사용할 profile 이름을 맞추세요.
# TODO 2. DB 접속 정보는 실제 값을 하드코딩하지 말고 환경변수로 분리하세요.
# TODO 3. 로컬과 운영 설정이 왜 달라져야 하는지 같이 확인해보세요.
```

또 아래 원칙을 반드시 지키세요.

* `TODO 1`, `TODO 2`, `TODO 3`처럼 번호를 붙인다.
* 한 TODO에는 한 동작만 넣는다.
* 문법 설명보다 흐름 설명을 적는다.
* 학생이 하지 말아야 할 것도 적는다.
* 운영 환경에서 왜 필요한지 드러나게 적는다.

예:

```text
# TODO: 민감한 값을 설정 파일에 직접 하드코딩하지 마세요.
# 이번 단계의 핵심은 '코드와 실행 환경 설정을 분리하는 감각'을 익히는 것입니다.
```

또는

```text
# TODO: 로그를 단순 출력이라고 생각하지 마세요.
# 서버에서는 '지금 무슨 일이 일어났는지' 확인하는 가장 기본 도구입니다.
```

## 14. starter 코드 파일 작성 규칙

starter 파일은 아래 성격을 가져야 합니다.

* 바로 실행은 가능해야 함
* 핵심 부분은 비어 있거나 TODO로 남아 있어야 함
* TODO는 학생이 수업 시간 안에 채울 수 있어야 함
* TODO 없는 보일러플레이트는 제공
* 로컬 또는 서버에서 실행 확인이 가능해야 함

필수 starter 파일 예시:

* `Dockerfile`
* `docker-compose.yml` 또는 `run-prod.sh`
* `src/main/resources/application-prod.yml` 또는 profile 예시
* `.env.example`

필요하면 추가 가능:

* `deploy.sh`
* `README-run.md`

단, 범위를 과하게 넓히지 마세요.
