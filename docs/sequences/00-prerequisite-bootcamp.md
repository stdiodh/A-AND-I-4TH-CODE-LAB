## 1. 이번 작업의 목표

이번 시퀀스의 목적은 스프링 코드 구현이 아니라, 이후 백엔드 실습에서 학생이 막히지 않도록
아래 선수지식을 먼저 맞추는 것입니다.

- HTTP 메서드와 상태 코드
- JSON 요청/응답
- Postman 사용
- Git/GitHub 기본 흐름
- DB 기초 개념 (table, row, column, PK)

즉, 이번 시퀀스는 CRUD, JPA, Security 같은 스프링 주제를 구현하는 단계가 아닙니다.
실제 애플리케이션 코드를 과하게 만들지 말고,
학생이 이후 실습을 따라갈 수 있도록 준비시키는 문서와 자료 구조를 만드는 데 집중하세요.

## 2. 이번 시퀀스에서 학생이 완성해야 하는 최종 상태

학생은 이 시퀀스를 마친 뒤 아래를 할 수 있어야 합니다.

1. 요청을 보내고 응답을 읽을 수 있다.
2. Postman으로 GET/POST 요청을 보낼 수 있다.
3. JSON body 값을 수정할 수 있다.
4. git clone / git checkout -b / git add / git commit 을 해볼 수 있다.
5. branch를 왜 만드는지 설명할 수 있다.
6. 테이블, row, column, PK를 설명할 수 있다.
