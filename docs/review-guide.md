# Review Guide

> 메인 README로 돌아가기: [README](../README.md)

이 문서는 4기 Code Lab 과제 리뷰 기준을 정리합니다.
리뷰의 목표는 정답 코드와 동일하게 만드는 것이 아니라, 학생이 학습 목표의 핵심 흐름을 직접 구현했는지 확인하는 것입니다.

## 리뷰 우선순위

1. 실행 가능성: README의 실행 명령과 테스트 명령이 통과하는가
2. 학습 목표: 이번 시퀀스의 핵심 개념이 코드에 드러나는가
3. 요청 흐름: Controller, Service, Repository 또는 해당 핵심 컴포넌트의 책임이 섞이지 않았는가
4. 실패 처리: Validation, 예외, 인증, 외부 연동 실패 같은 시퀀스 목표가 확인되는가
5. 테스트: 정상 케이스와 실패 케이스가 학습 목표를 검증하는가
6. 코드 스타일: 위 항목을 해치지 않는 범위에서만 정리합니다.

## diff 기반 피드백

학생 실습 후 아래 명령으로 기준 구현과 비교합니다.

```bash
git fetch origin
git diff NN-implementation..NN-answer
```

멘토는 diff를 볼 때 아래 순서로 판단합니다.

- answer와 다른 부분이 요구사항 누락인지 확인합니다.
- 다른 구현이더라도 실행 결과와 테스트가 맞으면 허용 가능한 차이인지 판단합니다.
- 코드 스타일 차이는 기능 흐름과 검증이 확인된 뒤에 다룹니다.
- 학생이 스스로 설명할 수 없는 코드는 다시 `implementation.md` 단계로 돌려 확인합니다.

## 답을 바로 주지 않는 방식

- "정답은 이 코드입니다"보다 "현재 단계에서 어떤 값을 만들어야 하나요?"라고 묻습니다.
- 파일 전체를 보여주기보다 TODO 번호와 관련 메서드만 보게 합니다.
- answer 브랜치는 마지막 비교 기준으로 사용하고, 첫 해결 수단으로 안내하지 않습니다.
- 실패한 테스트 이름과 expected/actual 차이를 먼저 읽게 합니다.

## 리뷰 완료 기준

- 학생이 오늘의 요청 흐름을 말로 설명할 수 있습니다.
- 실행 명령 또는 테스트 명령으로 결과를 확인했습니다.
- checklist의 학생 항목이 완료되었습니다.
- answer와의 차이가 남아 있다면 의도된 차이인지, 수정할 차이인지 기록했습니다.
