# Branch Policy

## 기본 원칙

- 모든 토픽 레포의 가이드 브랜치는 `main`입니다.
- 학생 시작 브랜치는 `NN-implementation`입니다.
- 참고 정답 브랜치는 `NN-answer`입니다.
- legacy `implementation` / `answer` 브랜치는 새 안내에 사용하지 않습니다.

## 운영 흐름

1. `main`에는 레포 소개, 시퀀스 목록, 브랜치 안내만 둡니다.
2. 학생에게는 `NN-implementation`을 먼저 안내합니다.
3. `NN-answer`는 막혔을 때 또는 실습 후 비교용으로 안내합니다.
4. default branch는 반드시 `main`으로 둡니다.

## Legacy 브랜치

legacy 브랜치가 남아 있으면 deprecated로 표시합니다.
삭제 여부는 수업 영향 확인 후 GitHub UI에서 수동으로 결정합니다.

현재 수동 조치 후보는 중앙 [README](../../README.md)의 `수동 GitHub 조치 목록`을 따릅니다.
