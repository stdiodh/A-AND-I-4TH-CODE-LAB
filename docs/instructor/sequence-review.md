# Sequence Review

시퀀스 리뷰는 학습 난이도, 문서 흐름, 브랜치 정책을 함께 확인합니다.

## 문서 리뷰

- [ ] 학생용 README가 시작 브랜치를 먼저 보여줍니다.
- [ ] 실행 방법과 테스트 방법이 분리되어 있습니다.
- [ ] `docs/theory.md`가 정의보다 실습 맥락을 먼저 설명합니다.
- [ ] `docs/implementation.md`가 학생이 손으로 할 순서를 보여줍니다.
- [ ] `docs/checklist.md`가 학생용과 강사용 항목을 분리합니다.

## 난이도 리뷰

- [ ] TODO가 핵심 파일에 집중되어 있습니다.
- [ ] 한 TODO가 한 동작만 요구합니다.
- [ ] 설정, 보일러플레이트, 긴 인프라 코드는 미리 제공됩니다.
- [ ] 수업 시간 안에 구현 가능한 범위입니다.

## 완료 기준

- [ ] `NN-implementation`은 실행 가능한 starter입니다.
- [ ] `NN-answer`는 비교 가능한 완성 흐름을 보여줍니다.
- [ ] `./gradlew test` 또는 해당 시퀀스의 검증 명령이 동작합니다.
- [ ] manifest의 status와 실제 상태가 어긋나지 않습니다.
