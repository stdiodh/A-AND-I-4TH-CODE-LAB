# Mentor Guide

> 메인 README로 돌아가기: [README](../README.md)

이 문서는 포트폴리오 증빙용 멘토 운영 요약입니다.
실제 세션 체크리스트는 [강사 체크리스트](./instructor/checklist.md)와 [시퀀스 리뷰](./instructor/sequence-review.md)를 함께 봅니다.

## 세션 전 준비

- 오늘 시퀀스 번호와 토픽 레포를 [manifest](./manifest/sequences.yml)에서 확인합니다.
- 학생 시작 브랜치가 `NN-implementation`인지 확인합니다.
- 토픽 레포 `main`이 안내 브랜치 역할을 하는지 확인합니다.
- 실행 명령, 테스트 명령, 필요한 Docker 서비스를 확인합니다.
- Visual Lab 허브와 시퀀스 상세 페이지가 열리는지 확인합니다.

## 세션 중 질문 대응 기준

- 먼저 학생이 어느 브랜치에서 작업 중인지 확인합니다.
- 정답 코드를 바로 보여주기보다 `README`, `implementation`, `checklist`의 해당 단계로 되돌립니다.
- 질문이 코드 오류라면 실행 로그, 테스트 실패 메시지, 요청 body, 인증 헤더 순서로 확인합니다.
- 질문이 개념 오류라면 `theory.md`의 현재 코드 흐름과 Visual Lab 흐름을 함께 보게 합니다.
- answer 브랜치는 막혔을 때의 비교 자료이지 첫 설명 자료가 아니라는 점을 유지합니다.

## 세션 후 회고

- 학생이 많이 막힌 TODO를 기록합니다.
- 문서 설명이 부족한 지점과 코드 TODO가 과한 지점을 분리합니다.
- 다음 세션 전에 README, implementation, checklist 중 어느 문서를 보강해야 하는지 결정합니다.
- answer와 학생 구현의 차이를 단순 정오답이 아니라 학습 목표와 연결해 정리합니다.

## 다음 기수 재사용 기준

- 브랜치 이름이 시퀀스 번호와 맞아야 합니다.
- 실습 시작 명령과 테스트 명령이 README에 있어야 합니다.
- 체크리스트는 학생용과 강사용이 분리되어 있어야 합니다.
- Visual Lab은 흐름을 복습하는 보조 자료로 유지하고 정답 코드를 과도하게 노출하지 않습니다.
